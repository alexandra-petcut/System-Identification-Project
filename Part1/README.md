## Project Overview

This project implements a **polynomial approximation system** to model an unknown nonlinear function using system identification techniques. Given noisy input-output data, we develop a polynomial model that accurately approximates the underlying function while avoiding both underfitting and overfitting.

---

## Problem Statement

### The Challenge

We are given:
- A dataset of **input-output pairs** where outputs are generated by an **unknown, nonlinear, static function** *f*
- The outputs are corrupted by **additive, zero-mean Gaussian noise**
- The function has **two input variables** (*x₁*, *x₂*) and **one output variable** (*y*)

### The Goal

Develop a **polynomial model approximator** *ĝ(x₁, x₂)* that:
1. Accurately fits the unknown function *f*
2. Generalizes well to new, unseen data
3. Balances model complexity with prediction accuracy

### Why This Matters

This problem represents a fundamental challenge in:
- **Machine Learning**: Function approximation and model selection
- **Control Systems**: Modeling dynamic systems from data
- **Data Science**: Regression and prediction with noisy observations

## Theoretical Background

### 1. Polynomial Approximation

A polynomial approximator of degree *m* for a function with two inputs has the form:

```
ĝ(x₁, x₂) = φ(x₁, x₂) · θ
```

Where:
- **φ** is the regressor vector containing polynomial terms
- **θ** is the parameter vector to be estimated

#### Polynomial Terms Structure

For degree *m*, we include all terms where the sum of exponents ≤ *m*:

**Degree m = 1:**
```
φ = [1, x₁, x₂]
θ = [θ₀, θ₁, θ₂]
ĝ(x₁, x₂) = θ₀ + θ₁x₁ + θ₂x₂
```

**Degree m = 2:**
```
φ = [1, x₁, x₂, x₁², x₂², x₁x₂]
ĝ(x₁, x₂) = θ₀ + θ₁x₁ + θ₂x₂ + θ₃x₁² + θ₄x₂² + θ₅x₁x₂
```

**Degree m = 3:**
```
φ = [1, x₁, x₂, x₁², x₂², x₁x₂, x₁³, x₂³, x₁²x₂, x₁x₂²]
```

The pattern continues: for each degree *m*, we include:
- **Powers of x₁**: x₁⁰, x₁¹, x₁², ..., x₁ᵐ
- **Powers of x₂**: x₂⁰, x₂¹, x₂², ..., x₂ᵐ
- **Cross-terms**: x₁ᵏx₂ˡ where k + l ≤ m

### 2. Linear Regression for Parameter Estimation

Although the polynomial is **nonlinear in the inputs** (x₁, x₂), it is **linear in the parameters** (θ). This crucial property allows us to use linear regression.

#### The Normal Equation

Given *N* data points, we construct:

```
Φ = [φ(x₁⁽¹⁾, x₂⁽¹⁾)]     Y = [y⁽¹⁾]
    [φ(x₁⁽²⁾, x₂⁽²⁾)]         [y⁽²⁾]
    [      ⋮       ]          [ ⋮ ]
    [φ(x₁⁽ᴺ⁾, x₂⁽ᴺ⁾)]         [y⁽ᴺ⁾]
```

The optimal parameters are found by minimizing the **least-squares error**:

```
θ = arg min ‖Φθ - Y‖²
```

**Solution (Normal Equation):**
```
θ = (ΦᵀΦ)⁻¹ΦᵀY
```

In MATLAB, this is efficiently computed using:
```matlab
theta = PHI \ Y;
```

### 3. Model Selection: The Bias-Variance Tradeoff

Choosing the right polynomial degree *m* is critical:

#### Underfitting (Degree Too Low)
- **High bias**: Model too simple to capture the function's complexity
- **High training error** (MSE_id)
- **High validation error** (MSE_val)
- Example: Using a line to fit a wavy surface

#### Optimal Fit (Just Right)
- **Balanced complexity**: Model captures the true function structure
- **Low training error**
- **Low validation error**
- Generalizes well to new data

#### Overfitting (Degree Too High)
- **High variance**: Model memorizes training noise
- **Very low training error** (approaching zero)
- **High validation error** (increases again)
- Poor generalization to new data

### 4. Mean Squared Error (MSE)

The MSE measures the average squared difference between predicted and true values:

```
MSE = (1/N) Σᵢ₌₁ᴺ (ŷᵢ - yᵢ)²
```

We compute MSE on two datasets:
- **MSE_id**: Measures how well the model fits the training data
- **MSE_val**: Measures how well the model generalizes to unseen data

**Key Principle:** The optimal degree minimizes **MSE_val**, not MSE_id!

## Data Structure

The project uses MATLAB `.mat` files containing two data structures:

### Identification Dataset (`id`)
Used for **training** - estimating the polynomial parameters

### Validation Dataset (`val`)
Used for **testing** - evaluating model performance on new data

### Structure Fields

Each dataset contains:

```matlab
id.X{1}    % Vector of x₁ coordinates (n₁ grid points)
id.X{2}    % Vector of x₂ coordinates (n₁ grid points)
id.Y       % Matrix of outputs, size n₁ × n₁
           % Y(i,j) = f(X{1}(i), X{2}(j)) + noise
```

**Example:**
- If `n₁ = 50`, there are 50 × 50 = 2,500 data points
- `X{1}` and `X{2}` define a grid in the input space
- `Y` contains the corresponding (noisy) output values

## Methodology

### Step 1: Generate Polynomial Regressors

For each degree *d* from 1 to *m*:

1. **Create exponent pairs** for all terms where k + l ≤ d
   ```matlab
   for total = 0:d
       for e1 = 0:total
           e2 = total - e1
           exps = [exps; e1 e2]  % Store [k, l] pairs
       end
   end
   ```

2. **Build regressor matrix Φ** for all data points
   ```matlab
   for each grid point (i,j):
       x1 = X{1}(i)
       x2 = X{2}(j)
       for each exponent pair [k, l]:
           φ(row, term) = x1^k * x2^l
       end
   ```

### Step 2: Parameter Identification (Training)

Using the **identification dataset**:

1. **Construct regressor matrix** Φ_id (N × P matrix)
   - N = total number of data points (n₁ × n₁)
   - P = number of polynomial terms for degree d

2. **Prepare output vector** Y_column (N × 1 vector)
   - Convert the Y matrix to a column vector

3. **Solve for parameters** using linear regression:
   ```matlab
   theta = PHI_id \ Y_column_vector
   ```

4. **Compute predictions**:
   ```matlab
   Y_approx_id = PHI_id * theta
   ```

5. **Calculate training error**:
   ```matlab
   MSE_id(d) = mean((Y_approx_id - Y_column_vector).^2)
   ```

### Step 3: Model Validation (Testing)

Using the **validation dataset** with the **same θ** from training:

1. **Construct regressor matrix** Φ_val for validation data

2. **Compute predictions** using trained parameters:
   ```matlab
   Y_approx_val = PHI_val * theta
   ```

3. **Calculate validation error**:
   ```matlab
   MSE_val(d) = mean((Y_approx_val - Y_val_column_vector).^2)
   ```

### Step 4: Select Optimal Degree

1. **Plot MSE_id and MSE_val** vs. polynomial degree

2. **Find degree that minimizes MSE_val**:
   ```matlab
   [min_MSE_val, optimal_degree] = min(mse_val)
   ```

3. **Store the best model** (parameters and predictions at optimal degree)

### Step 5: Visualize Results

1. **Plot MSE curves** showing underfitting/overfitting regions

2. **Create 3D surface plots** comparing:
   - True function values
   - Model approximation
   - For both ID and VAL datasets

3. **Overlay plots** (true in blue, approximation in red)

## Implementation Details

### Algorithm Flow

```matlab
1. Load data (proj_fit_11.mat)
2. Initialize variables (max degree m, MSE vectors)
3. FOR each degree d = 1 to m:
     a. Generate exponent pairs for degree d
     b. Build PHI_id and Y_column_vector for ID dataset
     c. Solve: theta = PHI_id \ Y_column_vector
     d. Compute: Y_approx_id = PHI_id * theta
     e. Calculate: MSE_id(d)
     f. Build PHI_val and Y_val_column_vector for VAL dataset
     g. Compute: Y_approx_val = PHI_val * theta
     h. Calculate: MSE_val(d)
     i. Track minimum MSE and corresponding approximations
4. END FOR
5. Plot MSE_id and MSE_val vs degree
6. Find optimal degree (minimum MSE_val)
7. Visualize results with 3D surface plots
```

### Key Implementation Choices

#### 1. Matrix Vectorization
- The output matrix Y (n × n) is converted to a column vector for regression
- This allows standard matrix operations: θ = Φ \ Y

#### 2. Grid-Based Data Structure
- Inputs form a regular grid: all combinations of X{1} and X{2}
- Enables efficient surface plotting with `meshgrid` and `surf`

#### 3. Efficient Parameter Computation
- Uses MATLAB's optimized backslash operator `\`
- Equivalent to solving: Φᵀ Φ θ = Φᵀ Y

#### 4. Tracking Optimal Models
- Stores best approximations during iteration
- Avoids re-computing after finding optimal degree

## Key Concepts

### 1. System Identification
The process of building mathematical models of dynamic systems from measured input-output data. This project focuses on **static nonlinear systems**.

### 2. Polynomial Regression
- **Classification**: Supervised learning, regression task
- **Model Type**: Parametric (fixed structure, learned parameters)
- **Linearity**: Linear in parameters, nonlinear in inputs
- **Solution**: Closed-form (normal equation)

### 3. Training vs Validation
- **Training (Identification)**: Learn parameters from data
- **Validation**: Test generalization on held-out data
- **Critical Rule**: NEVER use validation data for parameter estimation!

### 4. Model Complexity
- **Too Simple**: Cannot capture data patterns (high bias)
- **Too Complex**: Captures noise as patterns (high variance)
- **Just Right**: Captures true patterns, ignores noise

### 5. Least-Squares Regression
Minimizes the sum of squared residuals:
```
min Σ (yᵢ - ŷᵢ)²
```
- **Advantages**: Closed-form solution, robust, well-studied
- **Assumptions**: Errors are independent, normally distributed, constant variance

### 6. Cross-Validation
Using separate datasets for training and testing ensures:
- Unbiased performance estimates
- Detection of overfitting
- Better model selection
